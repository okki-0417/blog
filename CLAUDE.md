## ブログの方針

- エンジニアのキャリア像や成長について考えたことを発信する

## 開発規約

### コードのコメント

- 基本的にコメントでの補足はせず、コードに自明的に語らせること
- 「なぜ普通の実装をしないのか/なぜ自然な実装をしないのか」などの、Why Not については必要な場合のみ書くことができる

### CSS

- Tailwind CSS の w- や h- などのユーティリティにおいて、w-[100px] のような arbitrary value ではなく、w-25 のように、÷4 で割った数字で表現すること

### ブログコンテンツの管理

- ローカルのサンプル用の記事には、`sample-` のプレフィックスをつけること
- ローカルの下書き用の記事には、`draft-` のプレフィックスをつけること
- これらのプレフィックスがついた記事は、`.gitignore` に追加されているため、GitHub には公開されない
