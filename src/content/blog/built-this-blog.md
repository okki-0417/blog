技術記事は Qiita に書いているけれど、もう少しゆるく、自分の言葉で残したい場所がほしくなった。Qiita だと「役に立つ情報を書かなきゃ」と無意識にハードルを上げてしまう。日々の小さな気づきや、まだ答えが出ていない問いをそのまま置いておく場所として、個人ブログを立てることにした。

## 技術スタック

- **Astro** — Markdown 中心のサイトを作るならまず候補に挙がる。
- **Tailwind CSS v4** — ユーティリティで完結する手軽さ。
- **Cloudflare Pages** — 静的サイトの配信先として無料枠が広く、デプロイも GitHub と繋ぐだけ。

凝った機能は何も入れていない。ヘッダーすら無い。最初は `Home / Blog / About` とリンクを並べていたけれど、書いていくうちに「2 ページしかないサイトでナビゲーションを主張する意味があるか？」と疑問になり、削った。各ページのタイトル自体がそのページの宣言なので、それで十分だった。

## 書き方

`src/content/blog/` に Markdown を置くだけ。`title`、`description`、`pubDate` を frontmatter に書けば一覧に並ぶ。

```md
---
title: '記事のタイトル'
description: '概要を一文で。'
pubDate: 'Jun 02 2026'
---

本文。
```

push すると Cloudflare Pages が拾って自動でデプロイしてくれる。書く以外のことをほぼ考えなくていい構成にした。

## これから

ここに何を書くかはまだ決めていない。たぶん:

- 仕事や学習で詰まったこと
- 読んだ本や記事の感想
- うまく言語化できないけど考え続けていること

頻度は気にしないことにする。書きたいときに書く。
